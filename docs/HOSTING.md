# Как выложить ShlapaBank в интернет

Кратко: что нужно сделать, чтобы сайтом могли пользоваться из интернета.

---

## Платные и бесплатные варианты

| Вариант | Платно? | Комментарий |
|--------|---------|--------------|
| **Туннель (ngrok и т.п.)** | Бесплатный план есть | Лимиты по трафику и времени, ссылка меняется. Хватает для теста и показа друзьям. |
| **PaaS (Railway, Render, Fly.io)** | Часто есть бесплатный tier | Ограничения по часам работы и ресурсам. Для учебного/демо проекта обычно хватает. |
| **VPS (свой сервер)** | Платно | От ~100–200 ₽/мес (RU) или $4–6/мес (за рубежом). Бесплатных VPS почти не бывает. |

Итого: полностью **бесплатно** — туннель или бесплатный tier PaaS; **постоянно и стабильно** — платный VPS или платный план PaaS.

---

## Вариант 1. Быстрый доступ без своего сервера (туннель)

Подходит, чтобы дать ссылку друзьям или проверить работу с телефона. Компьютер с проектом должен быть включён и в сети.

### Шаги

1. Запустите проект локально:
   ```powershell
   cd C:\Users\V\IdeaProjects\shlapabank
   docker compose up --build
   ```

2. Установите **ngrok** (или аналог):
   - Скачайте: https://ngrok.com/download
   - Зарегистрируйтесь на ngrok.com и получите бесплатный токен.

3. В новом терминале запустите туннель на порт 8001:
   ```powershell
   ngrok http 8001
   ```

4. В консоли ngrok появится ссылка вида `https://xxxx-xx-xx.ngrok-free.app` — это и есть адрес вашего сайта в интернете. Отправьте её тому, кому нужно зайти.

**Минус:** пока вы не запустите `docker compose up` и `ngrok http 8001`, сайт недоступен.

## Вариант 2. Свой сервер (VPS) — сайт работает 24/7

Подходит, если нужен постоянный доступ по своему домену или IP.

### Что понадобится

- **VPS** (виртуальный сервер): например Timeweb, Selectel, Reg.ru, DigitalOcean, AWS и т.п.
- **Домен** (по желанию) или доступ по IP-адресу сервера.

### Шаги

1. **Создайте VPS** с Linux (Ubuntu 22.04 удобно). Запомните IP и пароль/ключ для SSH.

2. **Подключитесь по SSH:**
   ```bash
   ssh root@ВАШ_IP
   ```

3. **Установите Docker и Docker Compose** на сервер (инструкции есть на docker.com для Ubuntu).

4. **Скопируйте проект на сервер:**
   - Через Git: `git clone <ваш-репозиторий>` в папку на сервере,
   - или залейте папку проекта (например через FileZilla/SCP).

5. **На сервере создайте файл `.env`** (скопируйте из `.env.example` и поменяйте):
   - `SECRET_KEY` — длинная случайная строка (для продакшена обязательна).
   - Пароль БД и админа — не оставляйте дефолтные, если сервер доступен из интернета.

6. **Откройте порт 8001** в файрволе хостинга (панель VPS или `ufw`):
   ```bash
   ufw allow 8001
   ufw reload
   ```

7. **Запустите проект на сервере:**
   ```bash
   cd /путь/к/shlapabank
   docker compose up -d --build
   ```

8. Сайт будет доступен по адресу: `http://ВАШ_IP:8001` (или `http://ваш-домен.ru:8001`, если привязали домен к этому IP).

**Домен (по желанию):** в панели регистратора домена добавьте A-запись на IP вашего VPS. Тогда можно заходить по `http://ваш-домен.ru:8001`.

## Вариант 3. Облачный хостинг приложений (PaaS)

Сервисы вроде **Railway**, **Render**, **Fly.io** поднимают приложение из кода/репозитория и дают ссылку. Обычно нужно:

- Репозиторий на GitHub/GitLab.
- Подключить репозиторий к сервису.
- Указать, что приложение запускается через Docker (docker-compose или Dockerfile).
- Задать переменные окружения (SECRET_KEY, DATABASE_URL и т.д. — часто базу сервис создаёт сам).

Конкретные шаги смотрите в документации выбранного сервиса (Railway, Render и т.д.).

---

## Что обязательно сделать перед выходом в интернет

1. **Поменять `SECRET_KEY`** в `.env` на длинную случайную строку.
2. **Поменять пароль админа** (`DEFAULT_ADMIN_PASSWORD`) и пароль БД в `.env`.
3. Для постоянного хостинга (VPS/PaaS) рассмотреть **HTTPS** (например, Nginx + Let's Encrypt или встроенный SSL у PaaS).

---

## Кратко: что делать

| Цель                         | Действие                                      |
|-----------------------------|-----------------------------------------------|
| Показать друзьям на пару часов | Запустить `docker compose up`, затем `ngrok http 8001` |
| Сайт круглосуточно по своему IP/домену | Арендовать VPS, установить Docker, скопировать проект, настроить `.env`, открыть порт, запустить `docker compose up -d` |
| Без своего сервера, «как в облаке» | Выбрать PaaS (Railway/Render и др.), подключить репозиторий, настроить переменные и деплой |

После этого любой человек в интернете сможет открыть выданную ссылку и пользоваться сайтом.
